Simulation & Valuation

Monte Carlo Simulation

Simulating thousands of stock price paths using Geometric Brownian Motion to price options and understand uncertainty.

Why Monte Carlo?

Many derivatives are too complex for closed-form formulas like Black-Scholes. Monte Carlo simulation solves this by generating thousands of possible future price paths, computing the option payoff at the end of each, and averaging them (discounted to present value).

The law of large numbers guarantees that with enough paths, the average converges to the true expected value. This is how banks price exotic options, compute VaR, and run stress tests.

Real-world use: Beninga's Excel model (which you've seen in class) does exactly this — 10,000 GBM paths, compute payoff for each, discount average. The simulation below lets you see this live.

Geometric Brownian Motion (GBM)

Stock prices are modelled using GBM — continuous random walks with drift. The key insight is that log-returns are normally distributed, which ensures prices remain positive.

Continuous form: dS = μ·S·dt + σ·S·dW

Discrete simulation (Euler scheme):
St+Δt = St · exp[(μ − σ²/2)·Δt + σ·√Δt·Z]

where: Z ~ N(0,1) (standard normal random draw)
μ = drift (expected return), σ = volatility, Δt = time step
Drift (μ)
The expected daily/annual return of the stock. Under risk-neutral pricing (for option valuation), μ = r (risk-free rate), not the stock's actual expected return. This is a critical subtlety.
Random Shock (σ·Z)
Random component. Each step draws a new Z from N(0,1). This models the unpredictable daily noise in stock returns. σ·√Δt scales the shock to the appropriate time interval.
Risk-neutral pricing: When valuing derivatives via MC, we use μ = r (risk-free rate) as the drift, not the actual stock return. This is the same risk-neutral probability trick used in the binomial model. The discounted expected payoff is then the fair option price.

Convergence and Accuracy

Monte Carlo is subject to sampling error — with too few paths, the estimated option price can vary significantly between runs. More paths = better accuracy, but higher computation time.

Standard error of MC estimate = σpayoff / √N

95% confidence interval: MC price ± 1.96 × (σpayoff / √N)

To halve the error: multiply N by 4 (not 2)

Variance reduction: Banks use techniques like antithetic variates (pairing each Z with −Z), control variates, and quasi-random sequences to achieve BSM-level accuracy with fewer paths. With these methods, 10,000 paths can match what 1,00,000 naive paths achieve.

Practice Problem

Problem 5.1: A Monte Carlo simulation uses 1,000 paths. The standard deviation of payoffs across paths is ₹1,200. What is the standard error of the MC option price estimate (in ₹, round to 2 decimal places)?

SE = σpayoff / √N

Standard Error (₹)
Step-by-step:
SE = σpayoff / √N = 1,200 / √1,000
= 1,200 / 31.623 = ₹37.95

The 95% confidence interval for the option price would be: estimate ± 1.96 × 37.95 = estimate ± ₹74.4. A wide range — you'd need ~10× more paths (10,000) to get ±₹23.5.

Quick Check

1. When using Monte Carlo to price a European call option, the drift (μ) in the GBM model should be set to:

  • The stock's historical average return
  • The stock's beta × market risk premium
  • The risk-free rate (r) — risk-neutral pricing
  • Zero (prices are a random walk with no drift)

2. To reduce the standard error of a Monte Carlo estimate by half, you must:

  • Double the number of paths
  • Quadruple the number of paths
  • Use a finer time step
  • Increase volatility in the model

3. Monte Carlo simulation is most valuable (vs Black-Scholes) when:

  • Pricing a simple European call on a dividend-paying stock
  • The option has a very short time to expiry
  • Pricing path-dependent or exotic options where no closed-form exists
  • Implied volatility is constant across strikes
